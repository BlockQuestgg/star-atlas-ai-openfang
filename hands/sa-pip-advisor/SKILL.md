# Domain Expertise — DAO Governance Analysis

## POLIS Voting-Escrow Mechanics

POLIS is the governance token of the Star Atlas DAO:

- **Vote-escrow model** — lock POLIS for longer periods to gain more voting power
- **Longer lock = more weight** — incentivizes long-term alignment
- **Decay over time** — voting power decreases as lock period approaches expiry
- **Proposals (PIPs)** — Polity Improvement Proposals submitted by community members

## PIP Lifecycle

1. **Discussion** — community discussion on forums/Discord
2. **Draft** — formal PIP written with specific proposal details
3. **Review** — community review period
4. **Voting** — POLIS holders vote (quorum requirements apply)
5. **Execution** — if passed, proposal is implemented
6. **Post-mortem** — community evaluates outcomes

## Treasury Context

- DAO treasury holds ATLAS and other assets
- Treasury proposals affect the entire ecosystem
- Budget allocation requires careful analysis of ROI and risk
- Multi-sig execution for treasury movements

## Analysis Principles

### Neutrality
- Present facts and analysis, not opinions
- Use balanced language ("proponents argue... critics counter...")
- Acknowledge valid points on all sides
- Flag when analysis involves subjective judgment

### Rigor
- Cite specific PIP text, not paraphrases
- Quantify impacts where possible
- Identify assumptions explicitly
- Note what data would strengthen the analysis

### Accessibility
- Write for informed community members, not governance experts
- Define technical terms on first use
- Use tables and structured formats for complex comparisons
- Lead with the most important information

## Common Governance Risks

Watch for these patterns in PIP analysis:
- **Centralization risk** — does the proposal concentrate power?
- **Treasury depletion** — unsustainable spending proposals
- **Precedent setting** — what future proposals does this enable?
- **Implementation risk** — is the technical plan feasible?
- **Voter apathy** — does low turnout affect legitimacy?
- **Conflict of interest** — do proposers benefit disproportionately?
